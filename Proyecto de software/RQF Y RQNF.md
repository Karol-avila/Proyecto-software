# Requisitos Funcionales

| Código | Nombre | Descripción | Usuario Asociado |
|--------|--------|-------------|-----------------|
| RF001 | Prototipo de Registro y Acceso | El prototipo debe mostrar pantallas de registro y autenticación con validación básica (ejemplo: correo institucional). | Estudiantes universitarios |
| RF002 | Simulación de Subida de Archivos | El prototipo debe incluir la opción simulada de cargar materiales académicos, aunque sin almacenamiento real. | Estudiantes contribuyentes |
| RF003 | Flujo de Descarga/Visualización | El diseño debe permitir al usuario simular cómo descargar o visualizar archivos en línea. | Estudiantes buscadores de información |
| RF004 | Búsqueda y Filtrado | El prototipo debe mostrar un buscador con filtros (universidad, materia, tipo de recurso). | Estudiantes universitarios |
| RF005 | Muro de Publicaciones | El prototipo debe simular un feed donde los usuarios puedan publicar preguntas y respuestas. | Estudiantes colaboradores |
| RF006 | Intercambio de Habilidades | El prototipo debe incluir pantallas que simulen publicaciones o tutoriales compartidos entre estudiantes. | Estudiantes colaboradores |
| RF007 | Elementos de Reputación | El diseño debe ilustrar un sistema de puntos/insignias visible en perfiles de usuario. | Estudiantes contribuyentes |
| RF008 | Navegación Intuitiva | El prototipo debe ofrecer menús claros, rutas de usuario y jerarquía visual definida. | Estudiantes universitarios |
| RF009 | Prototipo Responsive | El diseño debe visualizarse tanto en formato escritorio como móvil. | - |

# Requisitos No Funcionales

| Código | Nombre | Descripción | Usuario Asociado |
|--------|--------|-------------|-----------------|
| RNF-01 | Usabilidad de Prototipo | El prototipo debe garantizar que las pantallas sean comprensibles y navegables sin instrucciones externas. | Estudiantes universitarios |
| RNF-02 | Coherencia Visual | El diseño debe mantener tipografías, colores y estilos consistentes en todas las pantallas. | Todos los usuarios |
| RNF-03 | Simulación Interactiva | El prototipo debe permitir interacciones simuladas (clics, navegación entre pantallas). | Estudiantes universitarios |
| RNF-04 | Accesibilidad en Diseño | Las pantallas deben cumplir principios de contraste, tamaño de letra legible e íconos descriptivos. | Estudiantes con diversidad funcional |
| RNF-05 | Documentación de Diseño | Todas las decisiones de diseño deben registrarse en Notion/Miro para asegurar trazabilidad. | Equipo de diseño |
| RNF-06 | Adaptabilidad | El prototipo debe mostrar versiones responsive para escritorio y móvil. | Estudiantes universitarios |
| RNF-07 | Claridad en Flujos | Cada flujo de usuario (registro, búsqueda, muro) debe estar claramente representado sin ambigüedad. | Estudiantes universitarios |
| RNF-08 | Feedback Visual | El prototipo debe incluir indicadores visuales de acción (ej: botones resaltados al hacer clic). | Estudiantes universitarios |
| RNF-09 | Consistencia con Metodología | El diseño debe alinearse con las fases de Design Thinking y Scrum, generando entregables validados en cada sprint. | Equipo de proyecto |

# Requisitos Funcionales del Sistema

| Código | Nombre | Descripción | Usuario Asociado |
|--------|--------|-------------|-----------------|
| RF001 | Registro y Autenticación | El sistema debe permitir que los estudiantes se registren y autentiquen mediante correo institucional o verificado. | Estudiantes universitarios |
| RF002 | Subida de Material Académico | El sistema debe permitir a los usuarios subir archivos (PDF, Word, imágenes) asociados a materias y universidades. | Estudiantes contribuyentes |
| RF003 | Descarga de Material Académico | El sistema debe permitir que los usuarios descarguen documentos de manera rápida y segura. | Estudiantes buscadores de información |
| RF004 | Clasificación y Búsqueda | El sistema debe contar con un módulo de búsqueda y filtrado por universidad, materia y tipo de documento. | Estudiantes universitarios |
| RF005 | Muro Social (Preguntas y Respuestas) | El sistema debe ofrecer un feed/muro para publicar preguntas, respuestas e ideas. | Estudiantes colaboradores |
| RF006 | Intercambio de Habilidades (Skills Exchange) | El sistema debe permitir a los usuarios publicar y acceder a habilidades o tutoriales compartidos. | Estudiantes colaboradores |
| RF007 | Gestión de Reputación | El sistema debe validar y reconocer la participación de los estudiantes mediante puntos o insignias. | Estudiantes contribuyentes |
| RF008 | Visualización de Archivos en Línea | El sistema debe permitir la visualización directa de PDFs sin necesidad de descarga. | Estudiantes buscadores de información |
| RF009 | Validación de Recursos | El sistema debe incluir un proceso básico de verificación (etiquetado de confiabilidad) de los recursos compartidos. | Comunidad académica |
